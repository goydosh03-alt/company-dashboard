Foundations

Principles and rules for the Pin-Up design system. Edited as Markdown via GitHub pull requests.

Principles

Consistency over novelty. Reuse tokens and components before creating new ones. Accessibility is a requirement, not a nice-to-have.

How to contribute

Open a pull request against the guide repository. Markdown files in src/content/guidelines render here automatically after deploy.
