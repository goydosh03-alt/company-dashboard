How this hub works

The whole guide lives in one GitHub repository and deploys itself. Here's the principle, so anyone on the team can understand and contribute.

1 · One source of truth

The repository holds the entire site — pages, styles and logic. Nothing lives on someone's laptop; the repo is always the up-to-date version everyone sees.

2 · It deploys itself

Every push to the main branch triggers Vercel to rebuild and publish automatically in about a minute. No manual deploy, no server to manage.

3 · Live data — no rebuild needed

Plugins (.zip files) and Figma file cards are uploaded right here in the app and stored in Vercel Blob. They appear for everyone instantly — the page reads them live, so you don't touch the repo to add or remove them.

4 · Editing text & guidelines

Written pages and guidelines are simple files in the repo. To change them, open a pull request on GitHub — after merge, the site rebuilds and shows the new version.
